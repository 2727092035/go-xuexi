# Day 1 笔记：变量与类型

## 今天先记住一句话

Go 是静态类型语言：每个变量都有确定类型，每种类型都有默认零值，编译器会阻止很多“不明确”的写法。

## 1. 变量是什么

变量就是给一个值起名字。Go 变量声明时，编译器必须知道它的类型。

最常用写法：

```go
name := "Tom"
age := 18
```

这里没有手写类型，但 Go 会根据右边的值推断：

- `"Tom"` 推断为 `string`
- `18` 推断为 `int`

`:=` 只能在函数内部使用，例如 `main` 函数里。

## 2. 三种变量声明方式

```go
name := "Tom"
```

适合函数内部，是日常最常见的写法。

```go
var name string = "Tom"
```

手动写出类型，适合你想强调类型，或者右边的值不够直观时。

```go
var name string
```

没有赋值时，Go 会给它一个零值。`string` 的零值是空字符串 `""`。

## 3. 零值是什么

Go 里没有 JS 那种到处可能出现的 `undefined`。变量只要声明了，就一定有值。

常见零值：

| 类型 | 零值 |
|------|------|
| `int` | `0` |
| `float64` | `0` |
| `string` | `""` |
| `bool` | `false` |

例子：

```go
var count int
var ok bool

fmt.Println(count) // 0
fmt.Println(ok)    // false
```

这个设计的好处是：变量状态更确定。代价是：有些时候你需要额外判断“这个 0 是默认值，还是业务上真的就是 0”。

## 4. 类型不会自动混用

Go 不会自动把 `int` 和 `float64` 混在一起计算。

```go
age := 18
price := 9.9

// fmt.Println(age + price) // 编译失败
fmt.Println(float64(age) + price)
```

你要自己决定怎么转换。这样啰嗦一点，但能避免隐式转换带来的隐藏 bug。

## 5. 常量 const

常量适合保存编译期就确定的值：

```go
const pi = 3.14
const appName = "Go Study"
```

不能把运行时才知道的结果赋给 `const`。

```go
// const now = time.Now() // 不行
```

## 6. fmt 常用占位符

`fmt.Printf` 用占位符控制输出格式：

```go
fmt.Printf("name=%s age=%d\n", name, age)
```

常用占位符：

| 占位符 | 含义 |
|--------|------|
| `%s` | 字符串 |
| `%d` | 整数 |
| `%f` | 浮点数 |
| `%.2f` | 保留两位小数 |
| `%v` | 按默认格式输出任意值 |
| `%T` | 输出变量类型 |

## 和 JS/TS 的关键差异

- Go 变量声明后必须使用，否则编译失败。
- Go 没有 `undefined`，每种类型都有零值。
- Go 不做隐式数字类型转换，`int` 和 `float64` 要显式转换。
- `:=` 很方便，但只能用在函数内部。

## 容易踩坑

- `:=` 左边至少要有一个新变量，否则会报错。
- `const` 只能保存编译期常量，不能保存函数返回值。
- `fmt.Println` 会自动换行，`fmt.Printf` 不会自动换行，需要自己写 `\n`。

## 练习时重点看什么

- 能不能分清 `var` 和 `:=` 的使用场景。
- 能不能说出 `int`、`string`、`bool` 的零值。
- 遇到类型不匹配时，能不能主动做显式转换。
